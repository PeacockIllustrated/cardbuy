## 03 · Schema · migration `0010_phase7_shop.sql`

### `lewis_listings`

Mirrors the Phase-1 `MockListing` shape exactly.

- `id` uuid pk
- `card_id` text (pokemontcg.io id, e.g. `base1-4`)
- `sku` text unique
- `variant` text `in ('raw','graded')`
- `condition`, `grading_company`, `grade` — same nullability rules as
  `lewis_binder_entries` (shape check constraint)
- `price_gbp` numeric(10,2)
- `cost_basis_gbp` numeric(10,2)
- `qty_in_stock` int ≥ 0
- `qty_reserved` int ≥ 0
- Available inventory is a generated column:
  `qty_available int generated always as (qty_in_stock - qty_reserved) stored`
  — cheaper than computing in every query, and the check constraint
  `qty_reserved ≤ qty_in_stock` prevents over-reservation.
- `status` text `in ('active','hidden','sold_out')`
- `is_featured` bool default false
- `featured_priority` int
- `condition_notes` text
- `created_at`, `updated_at`

**Indexes:** `(status)`, `(card_id)`, `(is_featured) where status='active'`.
**Touch trigger** on `updated_at`.

### `lewis_orders`

- `id` uuid pk
- `reference` text unique — `CB-ORD-YYYY-NNNNNN`, generated by
  `lewis_generate_order_ref()` + `lewis_order_ref_seq`.
- `buyer_id` uuid fk → `auth.users(id) on delete set null`
  (snapshot fields survive a deleted buyer account)
- `buyer_email`, `buyer_name` — snapshotted
- `status` text check — `pending_payment | paid | packing | shipped | delivered | refunded | cancelled`
- Money: `subtotal_gbp`, `shipping_gbp`, `total_gbp` numeric(10,2)
- `payment_method` text check `in ('stripe_card','paypal_in','stub')`
  (stub is the Phase-7 default until real payments ship)
- `shipping_method` text check `in ('royal_mail_tracked','royal_mail_special')`
- `shipping_address` jsonb — line1, line2?, city, postcode, country
- `tracking_number` text nullable
- `add_to_binder_opt_in` bool default true — the Slice B2 hook
- Lifecycle timestamps: `placed_at`, `paid_at`, `shipped_at`, `delivered_at`, `cancelled_at`
- `notes_internal`, `notes_buyer`
- `created_at`, `updated_at`

**Indexes:** `(buyer_id)`, `(status)`, `(created_at desc)`.

### `lewis_order_items`

- `id` uuid pk
- `order_id` uuid fk `on delete cascade`
- `listing_id` uuid fk `on delete restrict` — can't delete a listing
  someone bought
- Snapshotted: `card_id`, `card_name`, `set_name`, `variant`,
  `condition`/`grading_company`/`grade`
- `qty` int > 0
- `unit_price_gbp`, `line_total_gbp`
- `created_at`

## 04 · Atomic order creation · `lewis_create_order()` RPC

plpgsql function. The only way to create an order from application
code. Takes cart JSON, address, payment_method, opt_in. Does:

1. `begin` implicit transaction.
2. `select … for update` on every requested listing_id.
3. For each line, verify `qty_available >= requested`. Raise on fail.
4. Insert `lewis_orders`, then `lewis_order_items` (one per line).
5. For each listing, `update … set qty_reserved = qty_reserved + requested`.
6. Return the new order row.

Errors with a typed message string on availability failure so the JS
layer can surface "Out of stock — someone else grabbed this."

---

## 05 · Release-stock on admin cancel

Stock reservation is released by flipping `status → cancelled`. A
trigger on `lewis_orders` detects the status transition and
decrements `qty_reserved` across the order's items. Same trigger
handles `delivered` — that's where Slice B2's binder auto-add hooks
(or we run the auto-add in the server action; trigger is cleaner for
data integrity but awkward for cross-table RLS during insert).

**Decision:** put cancel-release logic in a trigger, binder auto-add
in the server action. Keeps the trigger surface minimal.

### `app/_actions/admin.ts` extensions

  (`deleteListing` = hard delete if no orders reference it, else flip to
  `status='hidden'`)
- `listAdminOrders(filters?)`
- `getAdminOrder(ref)`

  to `delivered`, if `add_to_binder_opt_in = true`, loop over items
  and call a private `addShopOrderToBinder(buyerId, item)` helper that
  inserts binder rows with `source='shop_order'` and
  `source_order_id = order.id`. Idempotent — if the order has already
  been delivered once, no-op.

## 11 · Slice B2 (piggyback)

Inside `updateOrderStatus(orderId, 'delivered')`:

```ts
if (newStatus === 'delivered' && order.add_to_binder_opt_in) {
  for (const item of items) {
    await addShopOrderToBinder({
      buyerId: order.buyer_id,
      orderId: order.id,
      cardId: item.card_id,
      variant: item.variant,
      condition: item.condition,
      gradingCompany: item.grading_company,
      grade: item.grade,
      quantity: item.qty,
    });
  }
}
```

`addShopOrderToBinder` is a private helper that upserts with
`source='shop_order'`. Idempotency: check if a row already exists with
`source_order_id = order.id AND card_id = item.card_id` (and tuple
matching). Skip if so.
